# Library Argument Ingestion

SciSiteForge should use Library-derived argument analysis as reviewed public
content and workbench assistance, not as raw automated publication.

## Public Surface

Public site output may include Library-derived material only when it is:

- public-safe under source restrictions;
- reviewed or explicitly labeled as a draft/workbench surface;
- source-anchored;
- correctly aligned to external claim taxonomies when such links are shown;
- scoped to the site mission and route class;
- excluded from private queues, model logs, and operational metadata.

## Workbench Surface

Workbench views may expose draft GroundRecall records, CiteGeist support gaps,
doclift chunk hints, and candidate Index to Creationist Claims alignments to
assist content review. These views must not be part of public Pages output
unless a site explicitly publishes a reviewed artifact.

## Study-Aid Surface

SciSiteForge can render reviewed study-aid pages when they serve the site
mission. The preferred structure is layered:

- at-a-glance orientation;
- source spine navigation;
- summary separated from analysis;
- glossary or concept links;
- worked critique/evidence examples;
- retrieval-practice or essay prompts;
- source-support and review status.

Public pages must not imply that a study aid replaces the original source. They
should link back to source records where available and show coverage gaps when
the aid is partial.

## Search

Public search corpora should index reviewed pages and public-safe argument
records. Workbench-only draft records, restricted Library material, raw model
outputs, and ingestion queues must remain outside public search manifests.
Study-aid records should be indexed by source, concept, claim, glossary term,
practice prompt, and worked-example topic only when they are reviewed and
public-safe.

## Site Use

Site-specific plans can use this shared contract to:

- identify duplicated static and dynamic content;
- compare claims against reviewed Library records;
- build reader paths from claims to evidence and critique;
- show reviewed Index to Creationist Claims alignments without forcing
  uncertain candidate matches into public navigation;
- generate bibliographies from reviewed support records;
- expose reviewed fallacy-pattern explainers and examples where they serve the
  site mission;
- flag pages needing updated citations or argument decomposition.
